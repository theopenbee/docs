---
title: 通过 IM 控制 Claude Code 进行项目开发
description: 使用专职数字员工，直接在 IM 应用中驱动 Claude Code 完成项目开发任务
---

本案例演示如何通过 IM 对话控制 Claude Code，让专职数字员工在指定项目目录中完成开发任务。

## 创建数字员工

打开 Web 控制台，进入**数字员工**页面。

点击**创建数字员工**，填写以下字段：

| 字段 | 值 |
|------|----|
| **名称** | 李白 |
| **描述** | 负责某某项目的开发 |
| **记忆** | *(留空)* |
| **工作目录** | `/path/to/your/project` |

**工作目录**是关键配置，它决定了 Claude Code 在哪个项目中执行任务。

也可以直接在 IM 应用中通过 Bee 创建：

> "创建一个叫李白的数字员工。描述：负责某某项目的开发。工作目录：/path/to/your/project"

## 工作原理

李白创建完成后，Bee 会根据描述将项目相关的消息路由给李白。李白在配置的工作目录中运行 Claude Code，并将执行结果回复给用户。

## 通过对话驱动开发

在 IM 应用中，发送如下消息：

> "李白，你给我看一下这个项目下用户认证模块的需求。"

> "李白，帮我实现一下登录接口。"

> "李白，跑一下测试，把结果告诉我。"

李白会在项目目录中通过 Claude Code 完成任务，并将结果反馈给你。

## 注意事项：将 `.openbee.md` 加入 `.gitignore`

OpenBee 会在数字员工的工作目录下自动生成 `.openbee.md` 文件，用于存储上下文和记忆。建议将此文件排除在版本控制之外：

```
echo ".openbee.md" >> /path/to/your/project/.gitignore
```

或手动添加到项目的 `.gitignore` 文件中：

```
# OpenBee 数字员工上下文
.openbee.md
```

## 使用建议

- **用项目名命名数字员工** — 清晰的名称（如"李白"、"后端开发"）有助于 Bee 准确路由消息。
- **在消息中直接指名数字员工**，确保消息由正确的员工处理：`"李白，现在认证模块进展怎么样了？"`
- **用记忆固化开发规范** — 例如将李白的记忆设为 `"实现功能时必须同步编写测试，遵循项目现有代码风格。"` 让开发规范自动生效。
- 可以为不同项目或同一项目的不同职责创建多个数字员工（例如一个负责前端、一个负责后端）。
