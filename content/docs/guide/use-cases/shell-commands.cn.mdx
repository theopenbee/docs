---
title: 通过 IM 执行命令
description: 使用专职数字员工，直接在 IM 应用中执行 Shell 命令
---

本案例演示如何通过 IM 直接向专职数字员工发送消息来执行 Shell 命令。

## 创建数字员工

打开 Web 控制台，进入**数字员工**页面。

点击**创建数字员工**，填写以下字段：

| 字段 | 值 |
|------|----|
| **名称** | Sham |
| **描述** | 如果发送的消息是一个命令，那就交给我来执行 |
| **记忆** | 执行用户发送的命令，并将结果发送给用户 |

也可以直接在 IM 应用中通过 Bee 创建：

> "创建一个叫 Sham 的数字员工。描述：如果发送的消息是一个命令，那就交给我来执行。记忆：执行用户发送的命令，并将结果发送给用户"

## 工作原理

Sham 创建完成后，Bee 会根据描述自动将命令类消息路由给 Sham。Sham 的记忆指示它执行命令并将输出结果回复给用户。

## 发送命令

在 IM 应用中，发送如下消息：

> `ls -la ~/Documents`

> `df -h`

> `git log --oneline -10`

Sham 会在其工作目录中执行命令，并返回输出结果。

## 使用建议

- 如需在特定目录中执行命令，可在创建数字员工时将 Sham 的**工作目录**设置为该路径。
- 可以在消息中直接指名 Sham，确保消息由它处理：`"Sham，ps aux | grep node"`
- 如需限制可执行的命令类型，可在 Sham 的记忆中添加约束，例如：`"只执行只读命令，禁止运行 rm、sudo 或任何破坏性操作。"`
