## 前置条件

- **Claude CLI** — Claude Code 数字员工所需。请参阅 [Claude Code 文档](https://docs.anthropic.com/en/docs/claude-code)了解安装方法。
- **Go 1.25+**（仅从源码构建时需要）
- **Node.js 20+**（仅从源码构建时需要）

## 通过 npm 安装

```bash
npm install -g openbee
```

## 通过脚本安装

**macOS / Linux：**

```bash
curl -fsSL https://raw.githubusercontent.com/theopenbee/openbee/main/install.sh | bash
```

**Windows (PowerShell)：**

```powershell
irm https://raw.githubusercontent.com/theopenbee/openbee/main/install.ps1 | iex
```

## 通过包管理器安装

**macOS (Homebrew)：**

```bash
brew install theopenbee/tap/openbee
```

**Windows (Scoop)：**

```bash
scoop bucket add theopenbee https://github.com/theopenbee/scoop-bucket.git
scoop install openbee
```

## 从源码构建

```bash
git clone https://github.com/theopenbee/openbee.git
cd openbee
make web    # 构建前端资源
make build  # 构建二进制文件
```

构建产物位于 `./bin/openbee`。

## 验证安装

```bash
openbee --version
```

你应该能看到版本号、提交哈希和构建日期。

## 常见问题

### Claude Code 不能以 root 用户运行

如果在启动 Claude Code 数字员工时看到以下报错：

```
dangerously-skip-permissions cannot be used with root/sudo privileges for security reasons
```

这是 Claude Code 的安全限制——它拒绝在 root 或 sudo 权限下使用 `--dangerously-skip-permissions` 运行。

**解决方案：** 使用非 root 用户运行 OpenBee。

```bash
# 创建专用用户（例如 openbee）
useradd -m openbee

# 切换到该用户并启动 OpenBee
su - openbee
openbee server -d
```

如果你已经以非 root 用户登录，请确保启动 OpenBee 时没有使用 `sudo`。
