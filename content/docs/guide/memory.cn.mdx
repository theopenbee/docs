---
title: 记忆
description: 通过对话管理全局记忆和数字员工记忆
---

记忆是 OpenBee 跨对话保留信息的方式。记忆分两种：**全局记忆**用于系统级知识，**数字员工记忆**用于单个数字员工的行为准则。它们让 Bee 和数字员工能够从交互中学习，并始终遵循你的偏好。

## 全局记忆

全局记忆存储长期知识，Bee 会在所有会话中使用它。包括用户偏好、工作流洞察和学习到的模式。

### 工作原理

每次你发送消息时，Bee 都会加载相关记忆来指导决策。当它发现值得记住的内容 — 一个偏好、一个模式、一条洞察 — 它会自动保存。

你也可以通过对话主动管理记忆：

### 保存记忆

> "记住我喜欢简洁的回复"

> "保存一条笔记：我们的 API 使用 snake_case 命名"

### 查看记忆

> "你记得关于我的什么？"

> "显示所有保存的记忆"

### 删除记忆

> "忘掉我的语言偏好"

> "删除关于 API 命名的记忆"

### 记忆结构

每条记忆包含以下字段：

| 字段 | 描述 |
|------|------|
| **键（Key）** | 描述性标识符（例如 `user_language_preference`） |
| **值（Value）** | 记忆内容 |
| **范围（Scope）** | `global` 表示全局，或会话标识符表示用户特定 |

Bee 使用描述性的键名作为自文档化标识，使记忆易于理解和管理。

### 全局 vs. 会话级

- **全局**记忆适用于所有用户和会话 — 适合通用洞察
- **会话级**记忆针对特定用户或对话 — 适合个人偏好

Bee 会根据上下文自动决定合适的范围。像"我在上海时区工作"这样的偏好是会话级的，而关于通用工作流模式的洞察可能是全局的。

## 数字员工记忆

数字员工记忆与全局记忆不同。它是每个数字员工的配置字段，定义持久的行为准则。可以把它理解为数字员工的"个性和规则"。

### 工作原理

数字员工记忆会写入数字员工的系统提示词，适用于该数字员工处理的每一个任务。它不会在对话之间改变 — 会一直持续到你主动更新为止。

### 设置数字员工记忆

在创建或更新数字员工时设置记忆：

> "创建一个叫 CodeBot 的数字员工，记忆是：始终遵循 PEP 8，优先使用类型注解，为每个函数编写测试"

> "更新 CodeBot 的记忆，加上：使用 pytest 而不是 unittest"

### 查看数字员工记忆

> "CodeBot 的记忆是什么？"

> "查看 CodeBot 的详细信息"

### 更新数字员工记忆

> "把 CodeBot 的记忆改为：专注于 Go 开发，遵循 Effective Go 指南"

> "给 CodeBot 的记忆补充：提交前必须运行 lint 检查"

### 数字员工记忆适合放什么

数字员工记忆适合定义以下内容：

**领域专业知识：**
> "你是一个经验丰富的 Python 开发者。始终遵循 PEP 8 规范，优先使用类型注解。"

**沟通风格：**
> "用简短的要点回复，避免长篇大论。"

**约束与安全：**
> "在没有和用户确认之前，绝不进行破坏性的 API 变更。"

**上下文与受众：**
> "你在帮助一个非技术用户，用简单易懂的方式解释一切。"

## 全局记忆 vs. 数字员工记忆

| | 全局记忆 | 数字员工记忆 |
|---|---|---|
| **范围** | 全局或按会话 | 每个数字员工独立 |
| **生命周期** | 永久，直到删除 | 跟随数字员工存在 |
| **谁读取** | Bee（用于决策） | 对应的数字员工 |
| **谁写入** | Bee（自动或按请求） | 你（通过创建/更新数字员工） |
| **用途** | 偏好、洞察、模式 | 行为规则、专业知识、约束 |
| **管理方式** | "记住……" / "忘掉……" | "创建/更新数字员工的记忆……" |

## 最佳实践

### 让 Bee 自然学习

Bee 在检测到偏好或模式时会主动保存记忆。你不需要明确告诉它记住每件事 — 自然交互即可，它会自行捕捉反复出现的主题。

### 对重要偏好使用显式记忆

对于你不想被遗漏的关键偏好，直接告诉 Bee：

> "记住：总是把代码审查任务分配给 CodeBot"

> "记住：我喜欢详细的解释，而不是简短的回答"

### 保持数字员工记忆的专注

数字员工记忆应简洁且与数字员工角色相关。避免塞入过多信息 — 聚焦于直接影响数字员工输出质量的规则和上下文。

### 定期审查和清理

随着时间推移，记忆可能会过时。定期检查已存储的内容并清理陈旧的条目：

> "显示所有记忆"

> "删除关于旧 API 端点的记忆"

### 用记忆代替重复指令

如果你发现自己在不同对话中反复发出相同的指令，把它保存为记忆。这样既节省 token，又能确保行为一致。
