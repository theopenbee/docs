---
title: 任务
description: 通过对话创建和管理任务
---

任务是分配给数字员工的工作单元。你可以通过在 IM 平台中向 Bee（协调器）发送消息来创建和管理任务，无需调用 API。

## 任务类型

OpenBee 支持三种任务类型：

| 类型 | 描述 | 示例 |
|------|------|------|
| **即时任务** | 数字员工可用时立即执行，这是默认类型。 | "助手，帮我写一份总结" |
| **倒计时任务** | 在指定时间执行。 | "下午3点提醒我参加会议" |
| **定时任务** | 按周期性计划执行（cron）。 | "每个工作日上午9点检查新的 issue 并汇总" |

## 创建任务

在 IM 对话中描述你的需求即可。Bee 会理解你的意图，选择合适的数字员工，并创建相应类型的任务。

### 即时任务

发送普通消息，Bee 会为匹配的数字员工创建即时任务：

> "助手，帮我总结一下今天的会议内容"

### 倒计时任务

包含具体时间，Bee 会创建倒计时任务：

> "下午3点提醒我提交报告"

> "5点半的时候，把今天的数据发给我"

### 定时任务

描述一个周期性计划，Bee 会创建基于 cron 的定时任务：

> "每个工作日上午9点，检查新的 issue 并汇总"

> "每小时获取一次系统状态并汇报"

定时任务内部使用标准 5 字段 cron 格式（`分钟 小时 日 月 星期`）。

## 数字员工选择

Bee 按以下优先级将任务路由到数字员工：

1. **明确指定** — 如果你在消息中提到了某个数字员工的名字，任务会分配给该数字员工
2. **对话延续** — 后续消息会分配给上一次任务的同一个数字员工
3. **描述匹配** — Bee 会将你的请求与数字员工的描述进行语义匹配
4. **兜底处理** — 如果没有匹配的数字员工，Bee 会建议你创建一个

## 查询任务

通过对话向 Bee 询问任务状态：

> "查看所有待处理的任务"

> "现在有哪些任务在运行？"

> "列出助手的定时任务"

## 取消任务

通过对话让 Bee 取消任务：

> "取消每日报告任务"

> "停止定期检查 issue 的任务"

## 任务生命周期

```
pending → running → completed
                  → failed
pending → cancelled
```

| 状态 | 含义 |
|------|------|
| `pending` | 任务已排队，等待被执行 |
| `running` | 任务正在被数字员工执行 |
| `completed` | 任务成功完成 |
| `failed` | 任务遇到错误 |
| `cancelled` | 任务在执行前被取消 |

定时（cron）任务在每次执行完成后会自动重新调度。
