---
title: 微信
description: 设置微信集成
---

## 前提条件

- 一个微信个人账号（用于扫码登录）
- 已安装 FFmpeg（语音消息转码所需）

## 通过交互式命令配置

设置微信最简单的方式是使用交互式配置命令：

```bash
openbee config
```

出现提示时，从平台列表中选择**微信**。向导会在终端中显示一个二维码——用手机微信扫描并确认即可完成认证。令牌和用户 ID 会自动填充。

如果扫码登录失败，你可以在提示时手动输入机器人令牌和用户 ID。

## 手动配置

在 `config.yaml` 中添加：

```yaml
bee:
  platforms:
    weixin:
      enabled: true
      token: "your-bot-token"
      user_id: "your-user-id"
      base_url: "https://ilinkai.weixin.qq.com"         # 可选
      cdn_base_url: "https://novac2c.cdn.weixin.qq.com/c2c"  # 可选
      max_media_size: 104857600  # 100MB，可选
```

| 字段 | 必填 | 描述 |
|------|------|------|
| `token` | 是 | 通过扫码登录获取的机器人令牌 |
| `user_id` | 是 | 通过扫码登录获取的用户 ID |
| `base_url` | 否 | 微信 API 基础地址（默认：`https://ilinkai.weixin.qq.com`） |
| `cdn_base_url` | 否 | 媒体 CDN 基础地址（默认：`https://novac2c.cdn.weixin.qq.com/c2c`） |
| `max_media_size` | 否 | 最大媒体文件大小，单位字节（默认：100MB） |

## 部署

重启 OpenBee。微信适配器使用长轮询方式接收消息，无需公网 URL 或 webhook。

## 媒体支持

微信支持图片、语音消息、文件和视频，大小不超过配置的 `max_media_size`（默认 100MB）。媒体在传输过程中使用 AES-128-ECB 加密。

语音消息以 SILK 格式接收，自动使用 FFmpeg 转码为 WAV 格式。请确保 FFmpeg 已安装且可访问（如果不在系统 PATH 中，请配置 `bee.media.ffmpeg_path`）。

## 会话过期

如果会话过期（错误码 -14），适配器会自动暂停 60 分钟后恢复轮询。如果令牌永久失效，你可能需要重新运行 `openbee config` 重新认证。
