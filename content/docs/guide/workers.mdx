Workers are Claude Code agents that execute tasks. Each worker has its own identity, persistent memory, and working directory.

## Worker Properties

| Property | Description |
|----------|-------------|
| **Name** | Display name for the worker |
| **Description** | What the worker does — used for task routing |
| **Memory** | Persistent context written to the worker's CLAUDE.md file |
| **Work Directory** | Filesystem path, defaults to `~/.openbee/worker/{id}`. Can be set to a custom absolute path. |
| **Status** | `idle`, `working`, or `error` |

## Creating a Worker

### Via Web Console

1. Open the web console at `http://localhost:8080`
2. Go to **Workers** → **Create Worker**
3. Fill in the name, description, and optional memory

## Managing Workers

Workers can be managed through the web console or by sending messages directly in your IM platform. For example, you can ask the bot to create a worker, update a worker's description, and more — all through natural conversation.

## CLAUDE.md Customization

Each worker has a `CLAUDE.md` file in its working directory that defines the worker's system prompt. This file is automatically managed by OpenBee based on the worker's memory field, but you can also edit it directly.

The working directory defaults to `~/.openbee/worker/{worker-id}/`. You can also specify a custom working directory (must be an absolute path) when creating or updating a worker.

## Worker Status

| Status | Meaning |
|--------|---------|
| `idle` | Worker is available to accept tasks |
| `working` | Worker is currently executing a task |
| `error` | Worker encountered an error during execution |
