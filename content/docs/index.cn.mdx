OpenBee 是一个数字员工解决方案，将 Claude Code 作为自主数字员工运行。每个数字员工都能够进行多步骤任务规划和独立执行，通过你现有的即时通讯平台进行沟通。

## 核心特性

- **AI 数字员工** — 具有持久记忆和 MCP 工具调用能力的 Claude Code 代理
- **多平台支持** — 飞书、钉钉、企业微信和 Telegram
- **任务调度** — 即时执行、倒计时和基于 cron 的定时任务
- **Web 控制台** — 管理数字员工、监控任务、查看执行日志
- **MCP 工具** — 可扩展的工具系统，增强数字员工能力
- **持久记忆** — 数字员工跨会话记住上下文

## 工作原理

OpenBee 采用三层架构，将通讯、调度和执行职责清晰分离：

```mermaid
graph TD
    subgraph IM["💬 IM 层（通讯层）"]
        A1["飞书"]
        A2["钉钉"]
        A3["企业微信"]
        A4["微信"]
        A5["Telegram"]
    end

    subgraph SCHED["📋 调度层"]
        B["任务调度与分发"]
    end

    subgraph EXEC["🤖 执行层"]
        C["数字员工\nClaude Code 代理"]
    end

    IM --> SCHED
    SCHED --> EXEC
    EXEC -. "回复结果" .-> IM
```

- **IM 层（通讯层）**：用户通过飞书、钉钉、企业微信、微信或 Telegram 发送消息，OpenBee 统一接入各平台的消息入口。
- **调度层**：负责接收来自 IM 层的消息，解析任务意图，将任务调度分发给合适的数字员工。
- **执行层**：数字员工接收任务后，使用 Claude Code 进行多步骤规划和独立执行，完成后将结果回传给用户。
