OpenBee 是一个数字员工解决方案，将 Claude Code 作为自主数字员工运行。每个数字员工都能够进行多步骤任务规划和独立执行，通过你现有的即时通讯平台进行沟通。

## 核心特性

- **AI 数字员工** — 具有持久记忆和 MCP 工具调用能力的 Claude Code 代理
- **多平台支持** — 飞书、钉钉、企业微信和 Telegram
- **任务调度** — 即时执行、倒计时和基于 cron 的定时任务
- **Web 控制台** — 管理数字员工、监控任务、查看执行日志
- **MCP 工具** — 可扩展的工具系统，增强数字员工能力
- **持久记忆** — 数字员工跨会话记住上下文

## 工作原理

OpenBee 采用三层架构，将通讯、调度和执行职责清晰分离：

```mermaid
graph TD
    subgraph IM["💬 IM 层（通讯层）"]
        A1["飞书"]
        A2["钉钉"]
        A3["企业微信"]
        A4["微信"]
        A5["Telegram"]
    end

    subgraph SCHED["📋 调度层"]
        B["任务调度与分发"]
    end

    subgraph EXEC["🤖 执行层"]
        C1["数字员工 1"]
        C2["数字员工 2"]
        C3["数字员工 N"]
    end

    IM --> SCHED
    SCHED --> EXEC
    SCHED -. "直接回复" .-> IM
    EXEC -. "回复结果" .-> IM
```

- **IM 层（通讯层）**：位于最顶层，是用户与系统交互的入口。用户通过飞书、钉钉、企业微信、微信或 Telegram 发送消息，消息向下传递给调度层处理。
- **调度层**：接收来自 IM 层的消息，解析任务意图后有两条路径：直接回复消息给 IM 层（如简单问答），或将任务调度分发给执行层的数字员工处理。调度层可同时管理多个数字员工，实现并行任务处理。
- **执行层**：由多个数字员工组成，每个员工都是独立的 Claude Code 代理，能够进行多步骤规划和自主执行。任务完成后，数字员工将结果直接回传给 IM 层，送达用户。
